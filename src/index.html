Sistema Integrado de Producción y Calidad


Contexto:

En planta se registraban datos de producción directamente desde sensores instalados en las máquinas inyectoras. Estos sensores enviaban archivos CSV con información operativa, como tiempos de ciclo, tiempos de parada y cantidad de piezas, que permitía calcular indicadores de rendimiento y disponibilidad.

Sin embargo, el sistema presentaba tres limitaciones críticas:

  • No registraba piezas no conformes, lo que impedía obtener el indicador de calidad y calcular el OEE completo.
  • Los sensores podían generar golpes fantasmas, inflando la cantidad real de piezas producidas.
  • La información permanecía aislada en la red local de planta, sin acceso para otras áreas.

Solución en Dos Etapas:

  • Una etapa para resolver las limitaciones operativas inmediatas.
  • Una etapa para robustecer la arquitectura desde un enfoque de ingeniería de datos.

ETAPA 1 : Incorporación de AppSheet para completar la información operativa

Acciones realizadas:

  • Desarrollo de un formulario en AppSheet para el registro manual de piezas no conformes.
  • Integración del formulario con Google Sheets para unificar información del sensor con la carga operativa.
  • Implementación de reglas de consistencia para corregir lecturas erróneas mediante el contraste producción esperada vs. real.

Impacto:

  • Incorporación del indicador de calidad de manera confiable.
  • Eliminación del sesgo por golpes fantasmas.
  • Obtención de un OEE completo (Disponibilidad, Rendimiento y Calidad).
  • Participación de los operarios en la mejora de la calidad del dato.

ETAPA 2 : Migración y fortalecimiento de la arquitectura con enfoque de Ingeniería de Datos

Una vez completada la captura integral de la información, se avanzó hacia una arquitectura más escalable, gobernada y apta para análisis corporativo.


Acciones realizadas / propuestas (versión robusta) :

  • Migración del proyecto a Google Cloud para liberar los datos del aislamiento de la red local.
  • Uso de Cloud Storage como repositorio de ingesta (raw) para los CSV de sensores.
  • Implementación de pipelines automáticos con Cloud Functions / Cloud Run para:
      • Procesar y limpiar datos (detección de golpes fantasmas, validación de rangos, normalización).
      • Unificar datos del sensor con datos ingresados desde AppSheet.
  • Almacenamiento central en BigQuery, estructurado en capas Bronze → Silver → Gold para garantizar trazabilidad y auditoría.
  • Integración con Looker Studio para dashboards empresariales basados directamente en BigQuery.

Impacto:

  • Datos accesibles y seguros para todas las áreas (Calidad, Producción, Ingeniería, Compras).
  • Eliminación de dependencias frágiles como hojas de cálculo manipuladas manualmente.
  • Datos trazables, auditables y consistentes (alineado a prácticas ISO 9001).
  • Automatización completa de ingesta, consolidación y validación.
  • Base sólida para futuros modelos predictivos (rechazos, anomalías de ciclo, mantenimiento).

Conclusión


Dividir la solución en dos etapas refleja una evolución clara:

Primero se resolvieron necesidades operativas críticas, y luego se escaló hacia una arquitectura profesional que garantiza calidad, disponibilidad y gobernanza del dato.

Esta estrategia permitió transformar un sistema aislado y limitado en una solución integral, automatizada y escalable para toda la organización.
